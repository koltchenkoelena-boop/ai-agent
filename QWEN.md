# Проект: multi-agent

Веб-чат для работы с несколькими LLM-агентами. Бэкенд на Python (FastAPI),
фронтенд — статические HTML-страницы в `web/`. Доступ только из сети Tailscale,
наружу в интернет не выставлен.

## Что где (карта файлов)

- `agent_server.py` — основной сервер агента (FastAPI). Запускается по ролям:
  planner / coder / researcher / reviewer (порты 8001–8004).
- `dispatcher.py` — диспетчер запросов между агентами.
- `session_db.py` — хранение сессий (SQLite через aiosqlite).
- `auth.py` — авторизация (JWT, bcrypt).
- `shared_storage.py` — общее хранилище состояния.
- `launcher.py` — оркестрация запуска.
- `run.sh` — локальный запуск без Docker (`./run.sh start|stop|status`).
- `docker-compose.yaml`, `Dockerfile`, `nginx.conf` — контейнерный вариант.
- `config.yaml` — конфиг ролей/моделей.
- `requirements.txt` — зависимости Python (fastapi, uvicorn, httpx, pyyaml,
  websockets, PyJWT, bcrypt, aiosqlite).
- `web/` — фронтенд (ТВОЯ основная зона работы):
  - `index.html` — главный чат-интерфейс (уже адаптирован под мобильные:
    бургер-меню `.menu-toggle`, выдвижной drawer `.sidebar.open` +
    `.sidebar-overlay`, `@media (max-width: 720px)`, touch-таргеты).
  - `dispatcher.html` — панель диспетчера.
  - `quad.html` — раскладка на 4 агента.

## Визуальный стиль (соблюдать)

Тёмная тема. Базовые цвета: `#0f3460`, `#1a1a4e`, акцент `#e94560`, текст `#eee`.

## Как ты (агент) работаешь

- Ты запущен в **изолированном git worktree** (`.qwen/worktrees/<session>/`)
  на своей ветке. Все правки делаешь здесь.
- Коммить **небольшими осмысленными коммитами** в свою ветку (один коммит =
  одна логическая правка) — так потом легко мёржить в `master`.
- **Не** трогай чужие ветки и worktree других агентов.
- Сообщения коммитов — на русском.

## Деплой и сервер — НЕ твоя зона

- Бэкенд продукта крутится на отдельном сервере (доступ по SSH/Tailscale).
- **Деплой на сервер и любую синхронизацию выполняет ЧЕЛОВEK, не ты.**
- НЕ запускай `ssh`, `scp`, `rsync`, `git push` и не пытайся подключаться
  к серверу. Твоя задача — менять файлы в worktree и коммитить в свою ветку.
- Python-venv намеренно отсутствует в git (`.gitignore`): это нормально,
  виртуалка пересоздаётся из `requirements.txt` и тебе для правки `web/*.html`
  не нужна.
